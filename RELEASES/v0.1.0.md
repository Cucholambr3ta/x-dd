# Release Notes — v0.1.0

> Notas user-facing. Primer release público de X-DD.

**Fecha:** 2026-05-30
**Anterior:** — (primer release)

## ✨ Novedades

- **Gated Pipeline de 6 fases** (Briefing → Spec → Plan → Build → QA → Retro) con
  checkpoint de aprobación humana en cada transición.
- **Gate keeper HMAC-SHA256** (`xdd-gate.py`): cada fase se firma criptográficamente;
  el gate detecta manipulación de artefactos. Incluye validación de **ejecución real de
  flujos** (`xdd-flow.py`), no solo presencia de archivos.
- **180 agentes** especializados con registry tipado.
- **7 IDEs** soportados por copia real (Claude Code, Cursor, opencode, Windsurf,
  VSCode/Copilot, Antigravity, Codex).
- **Instalable vía pip** (`pip install -e .`) con comando de terminal `xdd` (pipx) y
  entry-points por script — sin reescritura del tooling existente.
- **MockProvider determinista** (`xdd-provider.py`): testea flujos de agentes sin red ni
  tokens.

## 🛠 Mejoras

- **CI como ley**: pytest + bats + AgentShield corren como gate en cada PR.
- **AgentShield** (`xdd-shield.py`): audit estático de seguridad del propio framework.
- **VERSION** como fuente única de verdad de versión.
- **White-labeling** por proyecto (`xdd-brand.sh`).
- Observabilidad: spans OTel, session replay, cost tracker, context budget.

## ⚠️ Cambios incompatibles (breaking)

- Ninguno (primer release).

## 📚 Documentación

- Constitución v1.5 (9 artículos), 43+ ADRs, guías de instalación y por IDE.
- Guías de desarrollo interno reubicadas a `docs/dev/`.

## ⏳ Deprecaciones

- **MCP server** (`xdd-mcp-server`): deprecado, se elimina en v0.2.0 (ADR-0044). La copia
  real a IDEs lo reemplaza. Sigue funcional en v0.1.x.

## 🔒 Seguridad

- Gate key HMAC en `.xdd/.gate-key` (gitignored, nunca commiteado).
- 13 reglas de AgentShield sobre hooks, agentes y workflows.
