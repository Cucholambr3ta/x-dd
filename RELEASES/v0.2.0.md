# Release Notes — v0.2.0

> Notas user-facing. Release mayor — cero deuda técnica. Todos los stubs de v0.1.x implementados.

**Fecha:** 2026-05-31
**Anterior:** [v0.1.2](v0.1.2.md)

## ⚠️ Breaking Changes

**1. MCP server eliminado.** `xdd-mcp-server` ya no existe. Los 7 IDEs activan X-DD vía `/trigger` + `.agents/skills/` — sin servidor. Si tenías scripts que dependían de `xdd-mcp-install-global.sh`, actualízalos.

**2. Distribución híbrida.** `xdd init` ahora tiene dos modos:
- **Pip-mode** (recomendado): si tienes `pip install x-dd`, solo copia editables. El tooling viene del paquete.
- **Legacy**: comportamiento v0.1 (copia todo). Usar `--legacy` para forzarlo.

Para migrar proyectos v0.1: `xdd migrate run` (hace backup automático).

**3. Python 3.10+.** El piso baja de 3.12 a 3.10. Entornos que usan 3.10/3.11 ya son soportados.

## ✨ Novedades principales

- **Orchestration real**: `xdd orchestrate run --exec` ahora hace llamadas LLM reales (AnthropicProvider). Degrada limpiamente sin API key.
- **Observabilidad funcional**: 6 hooks OTel/cost activos (span-start/end, cost-record, budget-check).
- **Learning loop completo**: sprint tracking, pattern-extraction de traces, métricas de pipeline, temporal-awareness, judge-subagents AI.
- **`xdd update`** y **`xdd migrate`**: gestión del ciclo de vida del proyecto.
- **GitNexus opt-in** por defecto (licencia comercial). Activar: `XDD_GITNEXUS=1`.
- **Modo Base (<2 min)** documentado: pipeline completo sin MemPalace.
- Tabla comparativa honesta en README. Dogfooding visible.

## 🛠 Mejoras técnicas (no breaking)

- grader.json (borra parser YAML naive), make_parser() DRY en 12 scripts, pytest-cov, 23 tests core nuevos. 306 pytest, 146 bats, AgentShield 11 reglas (0 crit/high).

## 🔒 Seguridad

- GitNexus opt-in elimina invocación automática en uso comercial.
- AgentShield: reglas MCP eliminadas (ya no aplican); 11 reglas activas.
